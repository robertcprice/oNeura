# MC4D Gap Analysis

This note records the current gap between `oNeuro`'s native whole-cell path and the released `Minimal_Cell_4DWCM` implementation for `JCVI-syn3A`.

## Source Anchors

- Thornburg, Z. R., Maytin, A., Kwon, J., Solomon, K. V., et al. "Bringing the Genetically Minimal Cell to Life on a Computer in 4D." `Cell` (2026). DOI: `10.1016/j.cell.2026.02.009`
- MC4D GitHub repository: `https://github.com/Luthey-Schulten-Lab/Minimal_Cell_4DWCM`
- MC4D README: `https://raw.githubusercontent.com/Luthey-Schulten-Lab/Minimal_Cell_4DWCM/main/README.md`
- MC4D hook scheduler: `https://raw.githubusercontent.com/Luthey-Schulten-Lab/Minimal_Cell_4DWCM/main/Hook.py`
- MC4D launcher: `https://raw.githubusercontent.com/Luthey-Schulten-Lab/Minimal_Cell_4DWCM/main/Whole_Cell_Minimal_Cell.py`
- MC4D frozen release: `https://zenodo.org/records/15579159`

## What MC4D Already Has

- A Syn3A-specific multiscale orchestration stack that explicitly depends on `Lattice Microbes`, `odecell`, `btree_chromo` with `LAMMPS`, and `sc_chain_generation`, with optional `FreeDTS`.
- A single launcher, `Whole_Cell_Minimal_Cell.py`, that sets simulation cadence with `hook_step` and `write_step`, prepares geometry, initializes counts, and hands execution to the coupled solver stack.
- A hook scheduler in `Hook.py` that advances separate growth, global CME, DNA, and metabolism schedules through explicit time gates like `next_growth_time`, `next_gCME_time`, `next_DNA_time`, and `next_metabolism_time`.
- File-backed restart/output infrastructure around CME, flux, and restart directories, rather than only in-memory stepping.
- Organism-specific model content for the minimal cell, not only a generic substrate framework.

## What `oNeuro` Has Now

- A Rust-first native whole-cell runtime in `oneuro-metal` with RDME-like lattice diffusion, local chemistry microdomains, whole-cell scheduler/state logic, and GPU-backed hot paths.
- A substrate-first IR for reactions, assembly, localization, scalar process operators, affine reducers, subsystem signatures, and compiled coupling bundles.
- Descriptor-driven local atomistic refinement seeded from `oneuro-metal/specs/whole_cell_atomistic_templates.json` instead of synthetic clouds alone.
- A calibration path that fits descriptor-to-signature gains against local chemistry reports and MD probe sweeps.
- A bundled Syn3A organism descriptor with genome features, transcription units, composition pools, and geometry priors that now drive initialization, restart payloads, and stage-level process scaling in the native runtime.
- A persistent organism-expression layer that turns those descriptors into live transcription-unit activity state instead of treating them only as static startup metadata.
- Counted transcript/protein inventory state per transcription unit, so the organism-program path is starting to carry explicit evolving molecular inventories rather than only scale factors.
- A clean adapter boundary for the public MC4D stack, so we can reproduce their baseline separately while building a native engine.

## Path To Reach Their Level

1. Extend the current Rust-native Syn3A descriptor ingestion layer into curated genome-scale assets and restart imports that are closer to MC4D's published input ecosystem.
2. Replace remaining coarse chromosome/division approximations with dedicated native chromosome and septum mechanics.
3. Add restartable run manifests and artifact parity so long jobs can be resumed and audited like MC4D runs.
4. Fit and validate the native model against published Syn3A observables instead of relying mainly on internal consistency tests.

## Path To Surpass It

- Keep the multiscale structure, but compile more of it into Rust/GPU kernels instead of leaning on a mixed Python plus external-solver orchestration layer.
- Use descriptor-driven atomistic local refinement to update higher-scale reductions online, rather than freezing every mesoscale coupling by hand.
- Keep the substrate-first IR generic enough that the same stack can target cells, tissues, and synthetic molecular computers instead of only one curated organism.

## Non-Goals

- Claiming full all-atom whole-cell biological accuracy now.
- Treating the current native whole-cell runtime as already equivalent to the cited `Cell` result.
